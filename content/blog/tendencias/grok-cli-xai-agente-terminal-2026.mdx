xAI acaba de entrar al mercado de los agentes de codificación en terminal con **Grok Build**, su CLI para ingeniería de software. Al mismo tiempo, la comunidad de desarrolladores ya usa **Grok CLI** vía npm para automatizar tareas desde la consola con los modelos Grok. Aquí está lo que necesitas saber sobre ambas herramientas — y por qué este movimiento de xAI importa.

## El contexto: la guerra de los agentes CLI

Durante 2025 y 2026, el campo de batalla en herramientas de IA para desarrolladores se desplazó del editor de código al **terminal**. Anthropic lanzó Claude Code, GitHub expandió Copilot a modo agente, y OpenAI introdujo Codex CLI. El patrón es claro: los modelos más capaces ya no solo sugieren código — lo ejecutan, revisan repositorios, corren tests y hacen commits.

xAI llegó tarde al partido, pero con una propuesta técnica diferenciada: la arquitectura Heavy de múltiples agentes paralelos que ya usaba en Grok 4 aplicada ahora directamente al flujo de trabajo de un ingeniero de software en su terminal.

## Grok Build: el CLI oficial de xAI

**Grok Build** fue lanzado el 14 de mayo de 2026 como early beta exclusiva para suscriptores SuperGrok y X Premium Plus. Se instala con un solo comando:

```bash
# macOS / Linux
curl -fsSL https://x.ai/cli/install.sh | bash

# Windows PowerShell
irm https://x.ai/cli/install.ps1 | iex
```

El primer uso abre una autenticación vía navegador vinculada a la cuenta de X. No requiere configuración adicional — el agente ya tiene acceso al modelo `grok-build-0.1` disponible vía la API de Responses de xAI.

### El ciclo Plan-Revisar-Aprobar

La característica más diferenciadora de Grok Build es su flujo estructurado de ejecución. En lugar de modificar archivos inmediatamente, el agente:

1. **Analiza el repositorio** y genera un plan detallado con los cambios propuestos archivo por archivo
2. **Presenta cada modificación como diff** para revisión antes de ejecutar
3. **Espera aprobación** o comentarios del desarrollador antes de proceder

Esto resuelve uno de los mayores puntos de fricción con los agentes de codificación autónomos: la pérdida de control. El agente actúa, pero el desarrollador mantiene visibilidad y aprobación en cada paso.

<Callout type="tip">
  El comando `/plan` muestra el plan activo en cualquier momento de la sesión. `/rewind` permite retroceder el estado de la conversación si algo salió mal. `/fork` crea una rama alternativa sin perder el progreso anterior.
</Callout>

### Subagentes paralelos con aislamiento en worktrees

Para tareas complejas, Grok Build delega trabajo a **subagentes especializados** que operan en paralelo. Cada subagente trabaja en su propio worktree de git, lo que garantiza que los cambios experimentales no contaminen la rama principal.

Esta arquitectura es una extensión directa del modelo Grok 4 Heavy, que usa múltiples trayectorias de razonamiento en paralelo. Aplicado al desarrollo de software, el resultado es que el agente puede investigar, implementar y revisar cambios de manera simultánea en lugar de secuencial.

### Modo headless para CI y scripts

Grok Build incluye soporte para ejecución **sin interfaz interactiva** mediante el flag `-p`:

```bash
grok -p "Revisa todos los tests fallidos y propone un fix" --output json
```

Los formatos de salida soportados son `plain`, `json` y `streaming-json`, lo que facilita la integración en pipelines de CI/CD y workflows automatizados donde no hay un humano revisando en tiempo real.

<Callout type="warning">
  El flag `--always-approve` omite las confirmaciones interactivas y ejecuta los cambios de forma autónoma. Es útil para pipelines de CI, pero debe usarse con precaución en repositorios de producción — el agente puede modificar archivos sin revisión humana.
</Callout>

### Integraciones y extensibilidad

Grok Build se integra de forma nativa con el ecosistema MCP (Model Context Protocol), lo que permite conectarlo con herramientas externas:

- **Linear** para gestión de issues
- **Sentry** para monitoreo de errores
- **Grafana** para observabilidad

Además soporta archivos `AGENTS.md` para instrucciones a nivel de proyecto, plugins compartibles por equipos y el comando `/skillify`, que captura una sesión completa como skill reutilizable.

## Grok CLI (@vibe-kit/grok-cli): la alternativa open source

Paralelo al CLI oficial de xAI existe **Grok CLI**, un paquete npm open source desarrollado por Superagent bajo el nombre `@vibe-kit/grok-cli`. Su sitio oficial está en [grokcli.io](https://grokcli.io).

Esta herramienta apunta a un perfil diferente: desarrolladores que quieren interacción conversacional con archivos y proyectos desde la terminal, sin el peso de un CLI de empresa.

### Instalación

```bash
# Instalación global con npm
npm install -g @vibe-kit/grok-cli

# Requiere Node.js 16+ y una API key de xAI
export XAI_API_KEY="xai-..."
```

Una vez instalado, basta con ejecutar `grok` en el directorio del proyecto o `grok -d /ruta/al/proyecto` para especificar una carpeta.

### Qué puede hacer

Grok CLI ofrece una interfaz conversacional potenciada por **Grok-3** con las siguientes capacidades:

- **Operaciones con archivos:** el agente usa herramientas para ver, crear y editar archivos de manera automática según la instrucción del usuario
- **Integración bash:** permite ejecutar comandos de shell mediante lenguaje natural en la misma conversación
- **Selección automática de herramientas:** el modelo decide qué herramienta usar según el contexto, sin que el usuario tenga que especificarlo
- **Edición de alta velocidad:** modo opcional de edición a 4.500+ tokens/segundo con 98% de precisión
- **Soporte MCP:** extensión con servidores de Model Context Protocol (GitHub, Linear, etc.)

La interfaz está construida con **Ink**, la librería de React para terminales, lo que le da una apariencia visual más pulida que la mayoría de CLIs basados en texto plano.

<Callout type="tip">
  Grok CLI es la opción más accesible para empezar: no requiere suscripción premium, solo una API key de xAI. Ideal para proyectos personales, experimentación o equipos pequeños que quieren integrar Grok en su flujo de trabajo sin comprometerse con un plan de empresa.
</Callout>

## Comparativa: Grok Build vs Grok CLI vs Claude Code

| Característica | Grok Build | Grok CLI (vibe-kit) | Claude Code |
|---|---|---|---|
| Desarrollador | xAI (oficial) | Superagent (open source) | Anthropic (oficial) |
| Modelo base | grok-build-0.1 | Grok-3 | Claude (familia completa) |
| Modo plan | Sí (nativo) | No | Sí |
| Subagentes paralelos | Sí | No | Sí (herramientas) |
| Headless / CI | Sí (`-p` flag) | Limitado | Sí |
| Soporte MCP | Sí | Sí | Sí |
| Worktrees aislados | Sí | No | Sí |
| Instalación | `curl \| bash` | `npm install -g` | `npm install -g` |
| Acceso | SuperGrok / X Premium Plus | API key xAI | API key Anthropic |
| Precio mínimo | ~$40/mes (X Premium Plus) | Por uso (API) | Por uso (API) |

## Por qué importa este movimiento de xAI

xAI tiene un activo que ningún otro laboratorio de IA posee en este segmento: **integración directa con X (Twitter)**. Grok ya tiene acceso a datos en tiempo real de la plataforma, y es razonable esperar que Grok Build eventualmente pueda aprovechar esa integración — por ejemplo, monitorear issues mencionados en X o conectar contexto social con flujos de desarrollo.

La apuesta técnica también es sólida. El uso de worktrees de git para aislar subagentes paralelos es una arquitectura inteligente: permite explorar múltiples soluciones simultáneamente sin riesgo de conflictos, algo que los CLIs de un solo hilo no pueden hacer.

La limitación actual es el acceso. Restringir Grok Build a suscriptores premium durante la beta es una estrategia de lanzamiento conservadora, pero reduce la adopción inmediata entre desarrolladores independientes y equipos pequeños que quieren probar la herramienta antes de comprometerse.

## ¿Quién debería usar cada herramienta?

**Grok Build** es para equipos de ingeniería que trabajan en proyectos complejos, necesitan autonomía agéntica con revisión humana, y ya tienen o están evaluando una suscripción SuperGrok. Su flujo Plan-Revisar-Aprobar lo hace especialmente adecuado para entornos donde no se puede permitir que el agente cometa errores sin supervisión.

**Grok CLI (@vibe-kit/grok-cli)** es para desarrolladores individuales o equipos pequeños que quieren empezar a usar Grok en la terminal sin fricción. La instalación vía npm, el modelo conversacional y el soporte MCP lo hacen una entrada sólida para experimentar con agentes de codificación sin pagar una suscripción mensual fija.

Si ya usas **Claude Code** y estás satisfecho, no hay razón urgente para cambiar — Anthropic lleva más tiempo en este mercado y el ecosistema es más maduro. Pero si tu empresa está evaluando alternativas o quieres diversificar tu stack de herramientas de IA, Grok Build es la propuesta técnicamente más ambiciosa que xAI ha lanzado hasta la fecha.

<Callout type="tip">
  El espacio de los agentes CLI para desarrollo está evolucionando muy rápido. Hace doce meses estas herramientas no existían; hoy compiten activamente por convertirse en el copiloto estándar de los equipos de ingeniería. Evaluar opciones ahora, antes de que el mercado se consolide, da ventaja.
</Callout>
